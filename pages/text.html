Text Elements

Learn text formatting, emphasis, and content organization elements

Introduction

Text elements are the most fundamental building blocks of HTML content. They include paragraphs, emphasis elements, and various formatting tags that help structure and present text content.

<p> - Paragraph

The paragraph element defines a block of text. Browsers automatically add margins before and after paragraphs.

<p>This is a paragraph of text.</p>
<p>This is another paragraph.</p>

<strong> - Strong Importance

Indicates text of strong importance. Typically rendered as bold and important for screen readers.

<p>This is <strong>very important</strong> information.</p>

Result: This is very important information.

<em> - Emphasis

Marks text with emphasis. Typically rendered as italic and meaningful for screen readers.

<p>This is <em>emphasized</em> text.</p>

Result: This is emphasized text.

<span> - Generic Container

A generic inline container for styling or grouping text without semantic meaning.

<p>This text has <span style="color: red;">colored</span> content.</p>

<div> - Block Container

A generic block-level container used for grouping and styling content.

<div class="container">
  <h2>Section Title</h2>
  <p>Section content goes here.</p>
</div>

Other Common Text Elements

Element Purpose Example
<b> Bold text (no semantic meaning) <b>bold</b>
<i> Italic text (no semantic meaning) <i>italic</i>
<mark> Highlighted text <mark>highlighted</mark>
<small> Small text <small>small</small>
<del> Deleted text <del>removed</del>
<ins> Inserted text <ins>added</ins>
<sub> Subscript H<sub>2</sub>O
<sup> Superscript x<sup>2</sup>

Best Practices

  • Use <strong> instead of <b> for semantic importance
  • Use <em> instead of <i> for semantic emphasis
  • Keep paragraphs focused on a single idea
  • Use proper heading hierarchy
  • Don't use text elements for styling - use CSS instead

Accessibility Tips

  • Screen readers announce semantic elements differently
  • <strong> and <em> convey meaning beyond visual styling
  • Don't use <b> and <i> for semantic meaning
  • Ensure sufficient color contrast for visibility
