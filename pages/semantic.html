Semantic HTML5

Build accessible and meaningful web structures with semantic elements

What is Semantic HTML?

Semantic HTML uses elements that clearly describe their purpose and meaning to both browsers and developers. This improves accessibility, SEO, and code maintainability.

Structural Elements

ElementPurpose
<header>Introductory content or navigation
<nav>Navigation links
<main>Main content of the page
<article>Independent, self-contained content
<section>Thematic grouping of content
<aside>Sidebar or related content
<footer>Page footer

Layout Example

<!DOCTYPE html>
<html>
<body>
  <header>
    <h1>Site Title</h1>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <h2>Article Title</h2>
      <p>Article content...</p>
    </article>

    <aside>
      <h3>Related Links</h3>
    </aside>
  </main>

  <footer>
    <p>© 2026 Company</p>
  </footer>
</body>
</html>

Content Elements

ElementPurpose
<time>Represents a date/time
<mark>Highlighted or marked text
<code>Computer code
<kbd>User keyboard input
<var>Variable in mathematics/programming

Benefits of Semantic HTML

  • Accessibility: Screen readers understand page structure
  • SEO: Search engines better understand content
  • Maintainability: Clearer code for other developers
  • Mobile-friendly: Better rendering on all devices
  • Future-proof: Works with new technologies

Best Practices

  • Use <header> for introductory content
  • Use <nav> for navigation links
  • Use <main> for page's main content (only one per page)
  • Use <article> for standalone content
  • Use <section> for thematic grouping
  • Use <aside> for supplementary content
  • Use <footer> for page footer
  • Use proper heading hierarchy (h1-h6)

Common Mistakes

❌ Incorrect: Using div for everything

<div class="header">...</div>
<div class="main">...</div>
<div class="footer">...</div>

✅ Correct: Using semantic elements

<header>...</header>
<main>...</main>
<footer>...</footer>
