Forms & Input

Create interactive forms and collect user input effectively

Basic Form Structure

<form action="/submit" method="POST">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  
  <button type="submit">Submit</button>
</form>

Input Types

TypePurpose
textSingle line text input
passwordMasked password input
emailEmail address with validation
numberNumeric input
dateDate picker
checkboxMultiple choice
radioSingle choice from group
submitSubmit button
resetReset form
fileFile upload

Form Elements

<label> - Associates text with form elements

<label for="email">Email:</label>
<input type="email" id="email" name="email">

<textarea> - Multi-line text input

<textarea name="message" rows="5" cols="40"></textarea>

<select> - Dropdown list

<select name="country">
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
</select>

<fieldset> - Groups form elements

<fieldset>
  <legend>Personal Information</legend>
  <label><input type="checkbox"> Subscribe</label>
</fieldset>

Form Validation

  • required - Field must be filled
  • min/max - Number range validation
  • pattern - Custom validation pattern
  • type - Automatic validation (email, number, etc.)

Best Practices

  • Always associate labels with inputs
  • Use appropriate input types
  • Implement client-side validation
  • Always validate on the server too
  • Provide clear error messages
  • Make forms keyboard accessible
  • Use fieldsets for grouping related fields

Accessibility

  • Use label elements for all inputs
  • Use fieldset/legend for grouped fields
  • Provide clear instructions
  • Use ARIA attributes when needed
  • Ensure keyboard navigation works
  • Display error messages clearly
