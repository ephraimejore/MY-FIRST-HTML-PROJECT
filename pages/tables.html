Tables & Data

Master HTML table elements for presenting structured data

Table Elements

HTML provides semantic elements for creating well-structured data tables.

Basic Table Structure

<table>
  <thead>
    <tr>
      <th>Header 1</th>
      <th>Header 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Data 1</td>
      <td>Data 2</td>
    </tr>
  </tbody>
</table>

Table Elements Reference

ElementPurpose
<table>Container for table content
<thead>Groups header content
<tbody>Groups body content
<tfoot>Groups footer content
<tr>Table row
<th>Header cell
<td>Data cell

Complete Table Example

<table>
  <caption>Annual Sales Report</caption>
  <thead>
    <tr>
      <th>Quarter</th>
      <th>Revenue</th>
      <th>Growth</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Q1</td>
      <td>$100,000</td>
      <td>5%</td>
    </tr>
    <tr>
      <td>Q2</td>
      <td>$120,000</td>
      <td>20%</td>
    </tr>
  </tbody>
</table>

Colspan and Rowspan

Merge cells across multiple rows or columns:

<table>
  <tr>
    <td colspan="2">Spans 2 columns</td>
    <td>Normal cell</td>
  </tr>
  <tr>
    <td rowspan="2">Spans 2 rows</td>
    <td>Cell 1</td>
  </tr>
</table>

Accessibility Features

  • Use <caption> to describe table purpose
  • Use <th> for header cells with scope attribute
  • Use <thead>, <tbody>, <tfoot> for organization
  • Use headers attribute to associate cells with headers
  • Avoid using tables for layout - use CSS Grid instead

Best Practices

  • Keep tables simple and focused
  • Use descriptive headers
  • Avoid excessive colspan/rowspan
  • Make tables responsive for mobile
  • Use CSS for styling, not HTML attributes
